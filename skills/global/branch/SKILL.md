# Branch Skill

Parse a Jira ticket ID, create the branch following the project's naming convention, sync
dependencies, and surface any existing implementation plan.

---

## Gotchas

- If the branch already exists locally (`git checkout -b` fails with "already exists"), check it out with `git checkout <BRANCH>` instead — do not error out.
- If the branch exists on the remote but not locally, use `git checkout -b <BRANCH> origin/<BRANCH>` to track it.
- `git fetch origin` can fail if the remote is unreachable — note this to the user but proceed with local data.

---

## Step 1: Parse the Ticket ID

Extract the ticket ID from the user's message. It follows the pattern `[A-Z]+-[0-9]+` (e.g., `AIH-1234`, `MDR-567`).

If the user hasn't provided a ticket ID, ask for it.

---

## Step 2: Determine the Base Branch

Check which base branch to use:

```bash
git branch -a | grep -E '^\s*(remotes/origin/)?main$'
```

Fetch latest from remote first:

```bash
git fetch origin
```

---

## Step 3: Create the Branch

Follow the project's naming convention:

- **AIH repos**: branch name is the ticket ID only (e.g., `AIH-1234`)
  - Convention from CLAUDE.md: `Branch names: AIH-xxxx (ticket ID only, no description suffix)`
- **Other repos**: ask the user if unsure, or use `feature/<TICKET-ID>`

```bash
git checkout -b <TICKET-ID> origin/<base-branch>
```

---

## Step 4: Check for Existing Plan

Look for a plan artifact generated by the `/plan` skill:

```bash
cat .agents/artifacts/<TICKET-ID>-plan.md 2>/dev/null
```

- **If found**: read the plan and present a 2–3 sentence summary of the approach.
- **If not found**: offer to run `/plan <TICKET-ID>` to generate one before starting implementation.

---

## Step 5: Sync Dependencies

Detect the package manager and sync:

```bash
# Check which lock file exists
ls uv.lock poetry.lock Gemfile.lock 2>/dev/null
```

| Lock file | Command |
|-----------|---------|
| `uv.lock` | `uv sync` |
| `poetry.lock` | `poetry install` |
| `Gemfile.lock` | `bundle install` |
| None found | Skip, note to user |

---

## Step 6: Report

Summarize what was done:

```
## Branch Ready

- Branch: AIH-1234
- Base: main (origin/main @ abc1234)
- Deps: ✅ synced (uv sync)
- Plan: ✅ found at .agents/artifacts/AIH-1234-plan.md
  → "Add confidence field to the DuckDB export pipeline, propagating through the
     MongoDB projection, Polars schema, and ETL output."

Ready to start implementation.
```

Or if no plan was found:

```
- Plan: ⚠️  not found — run /plan AIH-1234 to generate one
```
